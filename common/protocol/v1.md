# WebSocket protocol v1

Desktop Pet Online v0.1 uses uppercase message types and top-level JSON fields.
The server owns user IDs and room routing; client-provided user IDs and room IDs
on chat or pet update messages are ignored.

WebSocket connections are accepted only on the `/ws` path.

## Client messages

```json
{"type":"LOGIN","username":"Alice"}
{"type":"ROOM_CREATE","roomId":"friends"}
{"type":"ROOM_JOIN","roomId":"friends"}
{"type":"CHAT","message":"Hello"}
{"type":"PET_UPDATE","state":{"mood":90,"hunger":30,"action":"happy","position":{"x":10,"y":20}}}
{"type":"HEARTBEAT"}
```

Limits: payload 16 KiB; nickname 24 characters; room ID 32 ASCII letters,
numbers, `_` or `-`; chat 500 characters; mood/hunger 0 to 100.

## Server messages

| Type | Purpose |
|---|---|
| `WELCOME` | Assigned user, pet and default room |
| `LOGIN_OK` | Normalized guest profile |
| `ONLINE_LIST` | Current connected users |
| `ROOM_LIST` | Active rooms |
| `ROOM_USERS` | Members of a room |
| `ROOM_JOINED` | Confirmation of active room |
| `CHAT` | Room chat event |
| `PET_UPDATE` | Room pet state event |
| `HEARTBEAT_ACK` | Application heartbeat response |
| `ERROR` | Stable error code and readable message |
